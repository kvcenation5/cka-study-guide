⚡ CKA Component Ports & Config Cheatsheet

Quick reference for exam — find correct ports in under 30 seconds

1. All Component Ports — Memorize These

Component Port Node Config File Location What It Does
kube-apiserver 6443 Control Plane /etc/kubernetes/manifests/kube-apiserver.yaml Main entry point — all kubectl commands hit this
etcd 2379 client   2380 peer Control Plane /etc/kubernetes/manifests/etcd.yaml 2379 = client access (backup/restore), 2380 = etcd peer sync
kube-scheduler 10259 Control Plane /etc/kubernetes/manifests/kube-scheduler.yaml Decides which node a pod runs on
kube-controller-manager 10257 Control Plane /etc/kubernetes/manifests/kube-controller-manager.yaml Runs control loops (deployments, nodes, endpoints)
kubelet 10250 All Nodes /var/lib/kubelet/config.yaml   /etc/systemd/system/kubelet.service.d/ Node agent — runs pods, reports status to API server
kube-proxy 10256 All Nodes DaemonSet — kubectl get cm -n kube-system kube-proxy Manages iptables/ipvs rules for service routing
NodePort range 30000–32767 All Nodes Set in kube-apiserver: --service-node-port-range External access range for NodePort services

2. Find the Correct Port — Fast (Under 30 Seconds)

🥇 Fastest — Check working kubeconfig

cat ~/.kube/config | grep server
# server: https://controlplane:6443  ✅

# Then compare against broken one
cat /root/CKA/admin.kubeconfig | grep server
# server: https://controlplane:4380  ❌

🥈 Check static pod manifest

grep -i "port\|bind" \
  /etc/kubernetes/manifests/kube-apiserver.yaml

# Look for:
# --secure-port=6443

🥉 Check what's actually listening

ss -tlnp | grep 6443
# or
netstat -tlnp | grep 6443
# confirms the port is live

Check running process flags

ps aux | grep kube-apiserver \
       | grep -i port
# shows all flags including port

3. Static Pod Manifests — Where Everything Lives

# All control plane components live here
/etc/kubernetes/manifests/
├── kube-apiserver.yaml       # API server config + port
├── etcd.yaml                 # etcd config + data dir + certs
├── kube-scheduler.yaml       # scheduler config
└── kube-controller-manager.yaml  # controller config + kubeconfig path

# kubelet is NOT a static pod — it's a systemd service
/var/lib/kubelet/config.yaml            # kubelet config
/etc/systemd/system/kubelet.service.d/  # kubelet service args

4. Component-Specific Port Commands

kube-apiserver

grep secure-port \
  /etc/kubernetes/manifests/kube-apiserver.yaml
# --secure-port=6443

# Check cert SANs too (useful for TLS issues)
grep tls-cert \
  /etc/kubernetes/manifests/kube-apiserver.yaml

etcd

grep -E "listen|port" \
  /etc/kubernetes/manifests/etcd.yaml
# --listen-client-urls=https://127.0.0.1:2379
# --listen-peer-urls=https://127.0.0.1:2380

# For backup — you need this cert path too
grep cert-file \
  /etc/kubernetes/manifests/etcd.yaml

kubelet

grep -i port \
  /var/lib/kubelet/config.yaml
# port: 10250

# Check kubelet is running
systemctl status kubelet
journalctl -u kubelet -n 20

kube-proxy

# kube-proxy config is in a ConfigMap
kubectl get cm kube-proxy \
  -n kube-system -o yaml | grep -i port

# Check the DaemonSet
kubectl get ds kube-proxy -n kube-system

5. Kubeconfig Troubleshooting — Exam Pattern

Common broken kubeconfig fields — check in this order

cat /path/to/broken.kubeconfig

# Field 1: server port (most common trap)
server: https://controlplane:4380  ❌ wrong port
server: https://controlplane:6443  ✅ correct

# Field 2: wrong hostname
server: https://127.0.0.1:6443    ❌ may not work externally
server: https://controlplane:6443  ✅ use hostname

# Field 3: wrong context name
current-context: wrong-context
kubectl config get-contexts          # list all contexts
kubectl config use-context kubernetes-admin@kubernetes
⚠️ Always test a kubeconfig before declaring it fixed:
kubectl get nodes --kubeconfig=/path/to/fixed.kubeconfig

6. One-Line Fix Commands

# Fix wrong port in kubeconfig
kubectl config set-cluster kubernetes \
  --server=https://controlplane:6443 \
  --kubeconfig=/root/CKA/admin.kubeconfig

# Or sed replace (fastest in exam)
sed -i 's/4380/6443/g' /root/CKA/admin.kubeconfig

# Verify fix
kubectl get nodes --kubeconfig=/root/CKA/admin.kubeconfig

7. etcd Backup — Port + Cert Quick Reference

# Get ALL etcd values you need from the manifest in one shot
grep -E "listen-client|cert-file|key-file|trusted-ca|data-dir" \
  /etc/kubernetes/manifests/etcd.yaml

# Output will give you exactly what to paste into backup command:
ETCDCTL_API=3 etcdctl snapshot save /opt/backup.db \
  --endpoints=https://127.0.0.1:2379 \
  --cacert=/etc/kubernetes/pki/etcd/ca.crt \
  --cert=/etc/kubernetes/pki/etcd/server.crt \
  --key=/etc/kubernetes/pki/etcd/server.key
💡 Tip: Run the grep first, copy the paths, then build the etcdctl command. Saves 2 minutes vs. typing paths from memory.
